# セミナー企画書: AIと一緒に「自分だけの業務ツール」を作ろう

## 基本情報

| 項目 | 内容 |
|------|------|
| タイトル | AIと一緒に「自分だけの業務ツール」を作ろう — LINE to Notion Inbox を題材に |
| 形式 | ハンズオン型ワークショップ |
| 所要時間 | 2時間（休憩含む） |
| 対象 | ノーコード/ローコードに興味がある人、日常業務を自動化したい人 |
| 前提スキル | プログラミング経験不問（LINEとNotionを使ったことがあれば十分） |
| 持ち物 | ノートPC、Googleアカウント、LINEアカウント、Notionアカウント |

---

## 企画趣旨

### 背景

- AIコーディングツール（Claude Code等）の登場で、「プログラミングできなくてもツールが作れる」時代になった
- しかし多くの人は「何を作ればいいか分からない」「AIに何を頼めばいいか分からない」で止まっている
- 実際に動くツールを一緒に作る体験を通じて、「自分にもできる」という実感を持ってもらう

### このセミナーで得られるもの

1. **動くツールが手に入る** — セミナー終了時にLINE→Notion自動連携が使える状態になる
2. **AIとの対話の仕方が分かる** — 何をどう伝えれば望む結果が出るかの感覚
3. **次に自分で作るための土台** — Google Apps Script + 外部API連携の基本パターン

## 伝えたいメッセージ

### メインメッセージ

> **「プログラミングは手段。AIが書いてくれる今、大事なのは "何を作るか" を考えること。」**

### サブメッセージ

- 完璧を目指さない。まず動くものを作って、使いながら直す
- 小さなツールでも、毎日使うなら価値がある
- 「自分の不便」は、自分にしか分からない。だから自分で作る意味がある

## 応用アイデア（セミナー内で紹介）

セミナーで作ったLINE→Notionの仕組みを応用すると、こんなこともできる:

| アイデア | 仕組み |
|---------|--------|
| 経費メモ | LINEにレシート写真+金額→Notion家計簿DBに登録 |
| 読書メモ | LINEに書名を送る→Notion読書DBに「読みたい」で登録 |
| 議事録 | LINEに要点を箇条書き→Notion議事録ページに自動整形 |
| 日報 | LINEに今日やったことを送る→Notion日報DBに日付付きで登録 |
| アイデアストック | 思いついたことをLINEに→Notionのアイデアボードに蓄積 |

## 運営メモ

### 事前準備

- [ ] 参加者にGoogleアカウント・LINE・Notionの事前登録を依頼
- [ ] GASプロジェクトのテンプレートを用意（clasp push済みの状態）
- [ ] スクリーン投影用のデモ環境を準備
- [ ] Wi-Fi環境の確認（参加者全員がWebアクセスできること）

### 想定されるトラブル

| トラブル | 対応 |
|---------|------|
| Notion Integrationの接続忘れ | DBページの「接続」から追加する手順を再案内 |
| GASデプロイ後のURL取得ミス | 「デプロイを管理」から最新URLを確認 |
| LINE Webhook検証でエラー | GASの仕様（初回リダイレクト）で正常。実メッセージで確認 |
| Notionトークンのコピーミス | 設定変更ダイアログから再入力可能 |

### 成功指標

- 参加者の80%以上が、セミナー終了時にLINE→Notionの連携が動作している状態
- 「自分でも何か作ってみたい」と思った人が50%以上（アンケートで計測）
