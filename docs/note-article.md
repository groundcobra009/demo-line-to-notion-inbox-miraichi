# LINEからNotionにタスクを投げ込む仕組みを作った話

## はじめに

「あ、あれやらなきゃ」

通勤中、買い物中、お風呂に入っているとき——ふとタスクを思いつく瞬間は、たいていスマホしか手元にありません。

私は普段、タスク管理にNotionを使っています。しかし、Notionアプリを開いて、データベースを探して、新しいページを作って、タイトルを入力して……という手順は、思いついた瞬間にサッとやるには重すぎます。

**「LINEにメッセージを送るだけでNotionのタスクデータベースに登録できたら最高じゃないか」**

そう思って作ったのが、この **LINE to Notion Inbox** です。

---

## 何ができるの？

LINE Botにメッセージを送ると、Notionのデータベースに自動でページが作られます。

<!-- 📸 スクショ①: LINEのトーク画面で「牛乳買う」と送信している様子 -->
<!-- 　 → LINEアプリのBot トーク画面をキャプチャ -->

```
📱 LINE で「牛乳買う」と送信
　　↓
📝 Notion のタスクデータベースに「牛乳買う」が登録される
```

<!-- 📸 スクショ②: Notionのデータベースに「牛乳買う」が登録された画面 -->
<!-- 　 → Notionのデータベースビューで、登録されたページが表示されている状態をキャプチャ -->
<!-- 　 → Status=Inbox, Source=LINE, CapturedAt=日時 が見えるとベスト -->

テキストだけでなく、**画像も送れます**。レシートの写真、ホワイトボードの写メ、気になったポスターなど、画像付きでNotionに記録できます。

<!-- 📸 スクショ③: LINEで画像を送信 → Notionに画像付きで登録された画面（Before/After並べると分かりやすい） -->

### 具体的にNotionに登録される情報

| プロパティ | 内容 |
|-----------|------|
| Name（タイトル） | 送ったメッセージの内容 |
| Status | 「Inbox」（自動設定） |
| Source | 「LINE」（自動設定） |
| CapturedAt | 送信した日時 |
| LineMessageId | 重複防止用のID |

つまり、送るだけで「未処理のInbox」としてNotionに溜まっていくので、あとでPCの前に座ったときにまとめて整理すればOKです。

## 仕組みの詳細

### 1. LINEからメッセージを受け取る

LINE Messaging APIのWebhook機能を使います。ユーザーがBotにメッセージを送ると、LINEのサーバーからGASのWebhookエンドポイント（`doPost`関数）にHTTPリクエストが飛んできます。

```
ユーザー → LINE Bot → LINE Server → GAS(doPost)
```

テキストメッセージと画像メッセージの2種類に対応しています。

#### GAS特有の制約

ここで一つ大きな壁がありました。LINE Webhookでは通常、`X-Line-Signature`ヘッダーでリクエストの署名検証を行います。しかし、**GASの`doPost()`ではHTTPヘッダーにアクセスできません**。

これはGASのプラットフォーム的な制約で回避不能です。そのため、WebhookのURLそのものを長くランダムなものにすることで、URLの推測を困難にするアプローチを取っています。

### 2. Notionにページを作成する

受け取ったメッセージをNotion APIを使ってデータベースに登録します。処理の流れは以下の通りです。

```
1. データベースの存在確認（なければ自動作成）
2. オプションプロパティの存在チェック
3. 重複チェック（同じLineMessageIdがないか確認）
4. 画像があればダウンロード＆Notionにアップロード
5. ページ構造の組み立て
6. Notion APIでページ作成
```

#### 重複防止の仕組み

LINEのWebhookはリトライされることがあります。同じメッセージが2回登録されるのを防ぐため、各LINEメッセージ固有の`MessageId`をNotionのプロパティに保存し、登録前に重複チェックを行っています。

#### 画像のアップロード

画像の処理には **Notion File Upload API** を使っています。

```
Step 1: アップロードセッション作成
  POST /v1/file_uploads → セッションIDを取得

Step 2: ファイル送信
  POST /v1/file_uploads/{id}/send → 画像データを送信

Step 3: ページ作成時に参照
  imageブロック → file_upload タイプで参照
```

当初はGoogle Driveに画像を保存してURLで参照する方法を試しましたが、権限管理やURL有効期限の問題があり、Notion純正のFile Upload APIに切り替えました。これにより、Notionが直接ファイルをホスティングしてくれるので、管理がシンプルになりました。

### 3. エラーログ

すべての操作はGoogleスプレッドシートの「ErrorLog」シートに記録されます。いつ、どんなメッセージが、どう処理されたかを後から確認できます。

<!-- 📸 スクショ⑤: ErrorLogシートの画面 -->
<!-- 　 → Timestamp, Level, Function, MessageID などのカラムが見える状態でキャプチャ -->
<!-- 　 → 数行のログが記録されている状態だとリアリティが出ます -->

## セットアップ方法

### 必要なもの

1. **Googleアカウント**
2. **Notionアカウント**（ワークスペースにインテグレーションを作成）
3. **LINE Developersアカウント**（Messaging API チャネルを作成）

### 手順の概要

#### ① LINE Bot を作る

1. [LINE Developers](https://developers.line.biz/) にログイン
2. 新しいMessaging APIチャネルを作成
3. チャネルアクセストークンを発行
4. チャネルシークレットを控えておく

<!-- 📸 スクショ⑥: LINE Developers コンソールの画面 -->
<!-- 　 → Messaging API チャネルの設定画面 -->
<!-- 　 → 「チャネルアクセストークン」の発行ボタン周辺をキャプチャ -->
<!-- 　 → トークンの値はモザイク or ダミーで隠す -->

#### ② Notion インテグレーションを作る

1. [Notion Integrations](https://www.notion.so/my-integrations) にアクセス
2. 新しいインテグレーションを作成
3. APIトークンを控えておく
4. タスクを入れたいデータベース（またはページ）にインテグレーションを接続

<!-- 📸 スクショ⑦: Notion インテグレーション作成画面 -->
<!-- 　 → 「New integration」ボタンを押した後の設定画面 -->
<!-- 　 → APIトークン（ntn_xxxxx）が表示されている部分をキャプチャ（値は隠す） -->

<!-- 📸 スクショ⑧: Notionデータベースにインテグレーションを接続する画面 -->
<!-- 　 → データベースの「…」メニュー → 「コネクトの追加」→ 作成したインテグレーションを選択 -->

#### ③ Google Apps Script をデプロイ

1. このリポジトリをクローン
2. `.clasp.json` にスクリプトIDを設定
3. `clasp push` でコードをGASにアップロード
4. GASエディタから「ウェブアプリとしてデプロイ」
5. スプレッドシートを開き、メニューからセットアップウィザードを実行
6. Notion・LINEの認証情報を入力

<!-- 📸 スクショ⑨: GASエディタのデプロイ画面 -->
<!-- 　 → 「デプロイ」→「新しいデプロイ」を選択した画面 -->

<!-- 📸 スクショ⑩: デプロイ後に発行されたWebアプリのURL画面 -->
<!-- 　 → URLが表示されている部分をキャプチャ（この URL を次のステップで使う） -->

#### ④ Webhook URLを設定

1. GASのデプロイで発行されたURLをコピー
2. LINE Developersのチャネル設定でWebhook URLに貼り付け
3. Webhookを有効化

<!-- 📸 スクショ⑪: LINE Developers の Webhook URL 設定画面 -->

## 工夫したポイント

### 1. プロパティのマッピングが柔軟

Notionのデータベースプロパティ名は人それぞれです。「Status」を「ステータス」にしている人もいるし、「状態」にしている人もいる。このツールでは、プロパティ名を自由にカスタマイズでき、存在しないプロパティはスキップする設計にしています。

```javascript
// Statusプロパティがあれば設定、なければスキップ
if (optionalProperties.status) {
  properties[mapping.status] = { select: { name: 'Inbox' } };
}
```

つまり、既存のデータベースをそのまま使えます。

### 2. データベースの自動作成

データベースIDを指定しなくても、親ページIDさえ指定すれば、必要なプロパティを持ったデータベースを自動で作成します。「とりあえず始めたい」人に優しい設計です。

### 3. GitHub Actions による自動デプロイ

`main`ブランチにpushするだけで、GitHub Actionsが自動的にGASにデプロイしてくれます。clasp（GAS用のCLIツール）を使った自動化です。Discordへの通知も設定できます。

<!-- 📸 スクショ⑭（任意）: GitHub Actionsのワークフロー実行画面 -->
<!-- 　 → デプロイ成功の緑チェックが表示されている状態 -->

### 4. テキストの自動分割

Notionのタイトルプロパティには2000文字の制限があります。長いメッセージを送った場合、自動的にチャンク分割して本文ブロックに格納します。

## 使ってみた感想

実際に運用してみると、**思いついたタスクをLINEに投げるだけ**という手軽さは想像以上に快適です。

- 買い物リストを思いついたらLINEに投げる
- 仕事のアイデアを思いついたらLINEに投げる
- 「あとで調べたいこと」をLINEに投げる
- 気になるポスターや看板を写真に撮ってLINEに投げる

全部Notionの「Inbox」に溜まるので、あとでPCの前で整理するだけ。GTD（Getting Things Done）でいう**「収集」のフェーズ**が圧倒的にラクになりました。

## 今後やりたいこと

- **複数データベース対応**: メッセージの接頭辞で送り先を振り分け（例：「買」で始まると買い物リスト、「仕」で始まると仕事タスク）
- **リッチテキスト対応**: LINEのテキスト装飾をNotionに反映
- **日次サマリー**: 1日の登録件数を毎晩LINEに通知
